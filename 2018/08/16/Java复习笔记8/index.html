进程: process
每一个进程，在操作系统中都有独立的内存空间，进程间互不干扰。
线程：thread
线程可以看成一条进程内存空间的访问路径。
当前的操作系统都是多任务，并发的操作系统，允许多任务同时执行，
可是CPU数量有限，这是就需要采用CPU分片的技术模拟实现多线程并发操作。
CPU任务分片时，需要不停的切换上下文，这比较耗效率。（线程池、NIO）
redis,node.js —–服务器进行单线程

注意：run()方法，无参，无返回值，无异常抛出，
      因为run()方法不是给编程人员调用的，是给JVM来调用的。
线程不能被强制结束
当run()方法结束后，自动结束。
线程中yield()与sleep()方法：
    yiled():使当前线程对CPU的占用率产生暂停，让别的线程有机会抢到CPU的使用权。
    sleep()：使当前线程的状态进入到阻塞状态，但为放弃CPU使用权
    join():在一个线程进入run()方法中，加入另一个线程，把当前线程置为阻塞状态，
            只有join的run()方法执行完之后，才会执行当前对象的run()方法。

锁：
synchronized:
1.synchronized锁方法：
当多个线程调用同一个对象的方法时，只能有一个线程进入这个方法体内，其他的线程在外面排队。
2.synchronized锁代码块：

    3.锁对象：
        Dog d = new Dog();
        synchronized(d){
            //给dog对象加锁后，其他线程都无法给dog再次加锁。 ---排他锁
            //意味着：在当前代码块中，可以多次调用dog中的方法，而不受其他线程影响。
        }
Lock:（接口）
    读写分离，写锁时排他的，读锁是共享的。
    方法：
        void lock() ；获取锁。  
        void unlock() ；释放锁。 
    ReadWriteLock（接口）：
        方法：
            readLock Lock readLock()；返回用于读取的锁 
            writeLock Lock writeLock()；返回用于写入的锁。
