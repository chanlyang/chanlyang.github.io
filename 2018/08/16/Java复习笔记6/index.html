java.util
Interface Collection
一组Object的集合
这个接口和子接口、子类都是集合
Collection集合可能是有序的，也可能是无序的。所以不能使用索引下标进行遍历。

boolean add(E e)添加
boolean remove(Objece o)移除
int size() 大小

java.lang
Interface Iterable 使用迭代器，对集合进行统一遍历

静态数组空间大小是一定的，最大的缺陷是数据是动态的，很难提前预估数组的大小。
静态数组的遍历方式是按照索引下标进行循环。

Set接口:不包含重复元素。
    实现类：
        HashSet:
            底层数据结构是HashMap，
            HashMap<k,v>,实际是把算有value置为空，而HashSet<E>存储的就是
                常用方法：
                    1.HashSet() 构建了一个新的，空的设置；支持HashMap实例具有默认初始容量（16）和负载因子（0.75）。
                    2.boolean add(E e) 如果没有已存在，则将指定的元素添加到该集合中。 
                    3.boolean contains(Object o)  如果这套 true返回包含指定的元素。 
                    4.boolean isEmpty()  返回 true如果这个集合不包含元素。 
                    5.Iterator<E> iterator()  返回此集合中元素的迭代器。 
                    6.boolean remove(Object o)  如果当前存在，则从该集合中移除指定的元素。 
                    7.int size()  返回该集合中元素个数（其基数）。 
        TreeSet:
            基于 TreeMap NavigableSet实施。用他们的 natural ordering元素是有序的，或由  Comparator在设定的创作时间，这取决于使用哪个构造函数。
                常用方法：
                    1.boolean add(E e)  如果没有已存在，则将指定的元素添加到该集合中。 
                    2.E ceiling(E e)  返回最小的元素在这组大于或等于给定的元素，或  null如果没有这样的元素。 
                    3.Comparator<? super E> comparator()  返回用于命令set中元素的比较，或 null如果使用此设置元素的 natural ordering。 
                    4.boolean contains(Object o)  如果这套 true返回包含指定的元素。 
                    5.Iterator<E> descendingIterator()  在这个集合中的元素上返回一个迭代器。 
                    6.NavigableSet<E> descendingSet()  返回此集合中包含的元素的反向顺序视图。 
                    7.E first()  返回当前在该集合中的第一个（最低的）元素。 
                    8.E floor(E e)  返回最大的元素在这组小于或等于给定的元素，或 null如果没有这样的元素。 
                    9.SortedSet<E> headSet(E toElement)  返回从这个集合的元素是严格小于 toElement部分。
                    10.NavigableSet<E> headSet(E toElement, boolean inclusive)  返回从这个集合的元素都小于（或等于部分，如果 inclusive是真的） toElement。
                    11.E higher(E e)  返回最小的元素在这套严格大于给定的元素，或 null如果没有这样的元素。 
                    12.Iterator<E> iterator()  在这个集合中的元素上返回一个迭代器。

如果没有已存在，则将指定的元素添加到该集合中。
Queue接口：具有先进先出的特点，不是有序集合
实现类：LinkedList，既是有序集合又是队列
LinkedBlockingQueue
重要方法：
boolean offer(); 插入
E poll(); 移除并返回
注意：Queue的父接口是Colloction，使用Queue的时候，最好使用offer()和poll()方法,不要使用Collection 的add()和remove()方法。
Map接口：
映射键到值的对象。一张Map不能包含重复的键，每个键可以映射到至多一个值。
常用方法：
Set<Map.Entry<K,V>> entrySet() 返回一个 Set视图的映射包含在这个Map。
V get(Object key) 返回指定的键映射的值，或 null如果这个Map不包含的键映射。
Set keySet() 返回一个 Set的关键视图包含在这个Map。
V put(K key, V value) 将指定的值与此映射中的指定键关联（可选操作）。
V remove(Object key) 如果存在（可选操作），则从该Map中移除一个键的映射。
default V replace(K key, V value) 仅当它当前映射到某一值时，替换指定的键的条目。
