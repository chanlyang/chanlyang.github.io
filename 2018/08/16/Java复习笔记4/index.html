访问修饰符:
public:对象和类，包内及包外
private:类内部
protected:包内及包外子类（主要用作子类调用父类属性），包内对象可以调，包外对象不能调
默认：类及包内（包内与public作用一致）

父类的构造：
Dog d1 = new Dog(“a”);
new Dog();会创建两块内存，先分配父类内存，然后分配子类内存。
父类如果有参构造，则参数必须由子类的构造函数传入。
Pet p1 = new Dog(“a”);
new Dog();一次分配了两块内存，p1中存的是父类内存的首地址。
Pet p1;表示dog只能指向父类的内存空间。
父类的引用要访问子类对象的内存，需要进行指针的强制转换。
父类与子类都用同名的属性：
父类引用指向父类属性。
子类引用指向子类属性。
重写：
父类和子类有一个同名、同参、同返回值的方法，则子类的方法的首地址会覆盖父类方法的首地址。
abstract类
抽象类中可以有抽象方法和非抽象方法
抽象类不能被new
抽象方法的主要目的就是要求子类重新自己的抽象方法。
优先使用组合，然后才是继承。
oop:规则
SRP:Single-Resposibility Principle单一职责原则
OCP:Open-Closed Principle开放封闭原则
对功能的扩展是open的
对程序代码的修改是close的
LSO:Liskvo-Substitution Principle里氏替换原则
ISP:Interface-Segregation Principle接口隔离原则
DIP:Dependecy-Inversion Principle依赖倒置原则
接口：
接口中没有属性，只有行为
接口中都是抽象方法
接口与抽象类的对比：
抽象类他是个Class，既有属性，又有行为
既有抽象方法，又有非抽象方法
一个类可以实现多个接口，但是只能有一个父类（类不能多重继承）————增加了灵活度
匈牙利命名法：例如m_strName
业务类，场景类。
