public class Dog{
public final int aa;
public static final int bb;
}
成员变量是final。则必须要在构造函数中初始化。
增加了static修饰符后，static与对象无关，则aa调用模式为Dog.bb。
这样就绕过了构造函数，则必须要初始化，不能在构造函数中初始化。

final修饰了一个引用类型，他锁定的是引用地址，不是应用对象中的内容
final Dog d2 = new Dog();
d2.name(“啊”);
d2 = d1; //报错。
public static final double PI = “3.14”;
编译器常量
总结：
final的特点：允许赋值一次，然后就不能修改
static的特点:和对象无关，允许通过类命直接调用
变量：生命期与作用域
局部变量：定义在函数体内的变量（基本类型、引用类型）
作用域：定义局部变量的{}范围内
生命期：
基本类型，当超过}。会被GC回收
引用类型，查看引用计数
局部变量是线程独享的模式，每个线程都有自己的内存空间
成员变量：
作用域：类内所有方法
生命期：
基本类型：属性和所在对象的生命期一致
引用类型：组合对象的生命期，可能是在对象外面创建的，和当前对象无关
很对对象的生命期，是由容器管理的
@Bean,Spring容器，javaEE容器(tomcat,weblogic)
static变量：
不能定义在方法内，只能作为成员变量存在
作用域：Dog.dd,相当于全局变量，可以在类的外部被大量调用
生命期：
static变量，可以被修改，从第一次调用才分配内存，直到程序结束，这块内存才被释放。
static变量存在JVM的方法区中
静态方法不能调用所在class的属性，属性是属于对象的，静态的是属于类的。
Dog d2; //stack中分配了一个4字节的空间，用于存放地址
d2 = new Dog();//在heap中分配内存，并返回这块内存的首地址
